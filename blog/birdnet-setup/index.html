Task 0: Acquire components

Task 1: Initial Raspberry Pi Zero 2 W set up

In this task, we‚Äôll install an operating system (OS) on the Raspberry Pi Zero 2 W, by loading it onto the microSD card.

  1. Plug the SD card writer into your computer, and put the microSD card into it.

  2. Install the Raspberry Pi Imager application on your computer, by selecting the correct ‚ÄúDownload for Windows/macOS‚Äù option on this page: Raspberry Pi Imager

  3. Open the Raspberry Pi Imager application, and make the following selections:

    • Raspberry Pi Device: Raspberry Pi Zero 2 W
    • Operating System: Raspberry Pi OS Lite (64-bit), ‚ÄúA port of Debian Bookworm with no desktop environment‚Äù.
      • This option may be nested under the ‚ÄúRaspberry Pi OS (other)‚Äù folder and require a little bit of searching.
    • Storage: Generic - SD/MicroSD Media, which is the microSD card that‚Äôs currently in the writer

    Your resultant selection screen should look like this:

    ‚Ä¶select NEXT!

  4. You‚Äôll be prompted to optionally ‚ÄúApply OS customization settings?‚Äù Select EDIT SETTINGS

    and then on the following page, set:

    • Hostname: birdnet-<pi_identifier>,
      • This will be the name of the Raspberry Pi, e.g., birdnet-foo, birdnet-backyard, birdnet-007, whatever! The identifier I chose in these screenshots is hannex, such that my full hostname is birdnet-hannex.
      • Make it something unique enough that if you set up multiple birdnets, you‚Äôll be able to keep them separate. I typically use the initials of the recipient.
      • Note that the angle brackets (‚Äú<‚Äù, ‚Äú>‚Äù) here and elsewhere in this tutorial should not be included in your chosen <pi_identifier>; they are simply convention used to indicate the user-specific parts of variables.
    • Username and password: <pi_username> / <pi_password>
      • This will be the username and password for the Raspberry Pi. I use birbwatcher as the username for all of mine, so that‚Äôs what you‚Äôll see in my screenshots. Make sure to write these values down somewhere, as you‚Äôll need them whenever you want to log into your Pi!
    • Configure wireless LAN: <your_current_wifi_network_name> / <your_current_wifi_network_password>
      • Important note 1: This must be the Wi-Fi network name (‚ÄúSSID‚Äù) and password for the Wi-Fi network your computer is currently using, and that you‚Äôll be using for the rest of the Pi setup. If you‚Äôre setting this device up for someone else, there‚Äôll be a separate step to additionally add their Wi-Fi network info later; for now, this should be yours.
      • Important note 2: If you have a 5 GHz option network (e.g., MyFunWifiNetwork and MyFunWifiNetwork_5G, do NOT use the 5G one! At time of writing, Raspberry Pi Zero 2 W‚Äôs do not support it.)
    • Make sure the time zone is correct for wherever your BirdNET-Pi will be installed.
    • Under the SERVICES tab, confirm that enable SSH and use password authentication are both selected‚Äîthey likely already are, by default.

    ‚Ä¶select SAVE!

  5. You‚Äôll be reprompted to ‚ÄúApply OS customization settings?‚Äù This time select YES. You might get a scary looking message asking you to confirm that any existing data on the card will be overwritten; select YES to continue. If prompted for a password, use the password for the computer you‚Äôre running the configuration from‚Äînot any of the passwords we‚Äôve now set for the Pi.

    The Raspberry Pi Imager will now proceed to write the operating system and configuration settings to your microSD card! It may take a few minutes, but the progress bar will tell you when its done, and the microSD card will be automatically disconnected from your computer (‚Äúejected‚Äù) once the writing is complete. At that point it will be safe to remove the card from the writer.

  6. Remove the microSD card from the writer and put it into the microSD card slot in the Raspberry Pi 2 Zero W.

  7. Connect the power cable to one of the Pi‚Äôs micro USB ports, and plug its other end into the wall. Once powered, the Pi will turn on automatically; you‚Äôll know the Pi is on because a green light will start flashing on the Pi. This may take a few moments.

Congratulations, you‚Äôve now installed the operating system on your Pi! When the Pi boots up, it will automatically use the Wi-Fi credentials you provided to connect to your wireless network. We can‚Äîand will!‚Äîdo the rest of the setup wirelessly.

Task 2: SSH time‚Äîtalk to the Pi

In this task, we‚Äôll ‚ÄúSSH onto the Pi‚Äù, i.e., send commands to it from another computer. It is barely worthy of its own section in this tutorial, but giving it one should make it easier to find again when your future self inevitably forgets the single command. :D

Why do we do this? In the previous task we chose the ‚ÄúLite‚Äù operating system for our Pi, which means we configured the Pi to run in ‚Äúheadless‚Äù mode‚Äîi.e., without a dedicated monitor, keyboard, or mouse. To interact with the Pi in the absence of those peripherals, we connect to it remotely over a wireless network, using a communication protocol called the Secure Shell Protocol (SSH).

Once your Pi is powered on, as long as your personal computer is on the same Wi-Fi network you configured the Pi to connect to you‚Äôll be able to SSH onto it from your personal computer. (I‚Äôve put that bit in bold because it is mission critical!)

When a step says ‚Äúrun‚Äù, that means to type or copy the given text into the terminal and then hit the Enter or Return key.

  1. SSH onto the device: on your personal computer, open up a terminal application. If you‚Äôre on a Mac, this is literally called Terminal; on Windows it‚Äôs called Windows Terminal. Run

    ssh <pi_username>@birdnet-<pi_identifier>.local

    To SSH onto the device I configured in the earlier screenshots, I do ssh birbwatcher@birdnet-hannex.local:

  2. If prompted to continue connecting, type yes and hit enter.

    • If you just hit enter without typing yes, you will not be connected, and it might get confusing. Try step 1 again, and this time type yes at the prompt!

  3. When prompted for a password, type the <pi_password> from above (without the angle brackets!!) and hit enter.

‚Ä¶that‚Äôs it, you‚Äôre on! Any commands you type into this terminal window will now be running on your Raspberry Pi. üòÆ If you want to close the SSH connection? Simply type exit and hit enter, or close the terminal‚Äôs application window by clicking on the X in the corner of the window.

If your SSH connection drops at any time‚Äîyour internet chokes, you decide it‚Äôs bedtime and want to finish another day, you accidentally close your terminal window, you intentionally close your terminal window‚Äîsimply SSH back on again.

  • Look at the start of each line: when you‚Äôre on the Pi, the ‚Äúprompt‚Äù‚Äîthe default text at the start of a line in your terminal‚Äîwill be <pi_username>@birdnet-<pi_identifier>.     In this screenshot, my local prompt is the path to my working directory (‚Äú~‚Äù) in a blue arrow, while the prompt on the Pi is the white birdnet-hannex:~/BirdNET-Pi$. The text styling will likely be different in your terminal, but the concept is the same.

    You can try out typing the same series of prompts as I did in this screenshot‚Äîanything on the blue or white lines‚Äîand you should see similar if not identical responses on your machines.

Task 3: BirdNET-Pi application installation

Okay. You‚Äôve set up the Pi, you‚Äôve ssh‚Äôd onto the Pi‚Äîit‚Äôs time to add the ‚ÄúBirdNET‚Äù part of BirdNET-Pi!

In this task, we‚Äôll download and run the installer for the BirdNET-Pi application and then change some of the Pi‚Äôs configuration for storage limits, restart behavior, and logging. Some of the following is going to seem a little silly, but it works. ¬Ø\_(„ÉÑ)_/¬Ø Bear with me!

If you‚Äôre prompted for a password at any point in this task, use the <pi_password> you set previously.

  1. In your remote terminal‚Äîthat is, a terminal window in which you‚Äôve SSH‚Äôd onto your Pi‚Äîrun

    sudo nano /etc/rc.local

    This will create a new file at path /etc/rc.local and open it in a program called nano, which is a text editor that runs in the terminal.

    Using your arrow keys and keyboard as with a regular text editor, Edit the file so its contents are exactly as follows:

    #!/bin/sh -e
iw wlan0 set power_save off
exit 0

    To do this you can either type these contents in manually, or copy and paste the contents using ctrl+c/ctrl+v commands, as with any other text editor.

    To exit this file, press ctrl and x at the same time; then y to save the modifications, then enter to select the path you originally opened as the save path. You will then be returned back to your command prompt.

    • Want to confirm that you wrote and saved the correct content to this file? Run

      cat /etc/rc.local

      to display the file contents in the terminal. If the displayed content matches the text above, you‚Äôre good to go‚Äîif it doesn‚Äôt, simply do sudo nano /etc/rc.local to reopen the file in your editor and try again!

  2. To make the file you just created executable, i.e., to enable the computer to run it, run

    sudo chmod u+x /etc/rc.local

    in the terminal.

  3. To update the Pi‚Äôs swap space limits, run

    sudo nano /etc/dphys-swapfile

    to open the file /etc/dphys-swapfile in nano.

    1. Use the down arrow to scroll through the file to the line that says CONF_SWAPSIZE=512 and edit it to be CONF_SWAPSIZE=2048

    2. Scroll down to the line that reads #CONF_MAXSWAP=2048 and edit it to be CONF_MAXSWAP=4096 (Note the lack of # at the beginning of the line!!).

    Your resultant file should look like this:

    A # at the start of the line means that the line is commented out‚Äîwhich means that the changes on that line will be ignored. We do not want them to be ignored, hence deleting the relevant #!

    To save the changes, press ctrl and x together, then y, followed by enter; once again you‚Äôll be returned to the terminal.

  4. In the terminal, run sudo reboot. This will cause the Pi to‚Äîyou guessed it‚Äîreboot, which will automatically close your SSH connection.

  5. Wait for a moment or two for the Pi to reboot, then SSH back onto it:

    ssh <pi_username>@birdnet-<pi_identifier>.local

    (Look familiar? :) )

  6. Okay, this is the bit that is going to feel silly. Just go with it. In your remote terminal, start the BirdNET-Pi installation script by running

    curl -s https://raw.githubusercontent.com/Nachtzuster/BirdNET-Pi/main/newinstaller.sh | bash

    A bunch of text will print out to the screen, as various files and dependencies are copied onto the Pi. You don‚Äôt have to pay attention to that text‚Äîyou just have to wait and wait, until the installation fails. This may take a long time, on the order of tens of minutes.

    That‚Äôs right: eventually, the installation script you just ran will fail(!) with a ‚ÄúNo space left on device‚Äù error. That‚Äôs expected! You want it to happen!2

  7. Okay, let‚Äôs fix the out-of-space error. Run the following:

    sudo mount -o remount,size=5G /tmp/

  8. Now re-run the installation step, first deleting the failed installation directory:

    rm -rf BirdNET-Pi/
curl -s https://raw.githubusercontent.com/Nachtzuster/BirdNET-Pi/main/newinstaller.sh | bash

    This step is slow, as lots of things happen during installation. Be patient!

    This time the installation script should succeed. You‚Äôll know it has succeeded because the Pi will reboot, which will close your SSH connection.

    If it does NOT succeed, i.e., you still get a failure message when you run that curl command, repeat the commands in this step. Try this a couple times before giving up and emailing me that my instructions did not work for you!3

  9. SSH back on to your Pi again! I bet you remember the command this time.

    You can also probably use the up-arrows in your terminal to page back through the previous terminal command history, which at this point definitely contains your SSH command! You can hit enter on any item in the command history to rerun it.

  10. Now edit the amount of storage space the system logs are allowed to use: run

    sudo nano /etc/systemd/journald.conf

    which will dump you into yet another file in your text editor!

    Use the arrow keys to scroll down to the line that says #SystemMaxUse= and remove the # to change the line to be SystemMaxUse=200M

    and then save the file: ctrl+x , Y, enter. You‚Äôre now a pro at this in-terminal text editor stuff!

  11. Restart the logging with those new settings by running

    sudo systemctl restart systemd-journald

Congrats! You‚Äôve now installed the BirdNET-Pi application! üê¶‚Äç‚¨õ The finicky part is over.

Task 4: Configure the BirdNET-Pi application

For the following steps, use whatever browser on your computer you prefer‚ÄîChrome, Firefox, whatever. Task 5 (Optional): Install Tailscale for remote monitoring

If you‚Äôre gifting this BirdNET-Pi to someone else who wants you to act as tech support for their installation, or you want to be able to check on your own installation while you‚Äôre not connected to your home Wi-Fi network, you can add your Pi to a Tailscale network‚Äîbasically, it acts as a virtual private network for your various devices.

Click to see instructions

  1. Sign up for a Tailscale account, if you don‚Äôt already have one.

    You do not need to do anything other than choose (or make) an account to sign in with. You don‚Äôt need to bother following their steps to ‚Äúadd a first device‚Äù unless you want to, you can do it later.

  2. In your terminal, SSH onto the Pi and install Tailscale by running

    sudo apt-get install apt-transport-https

curl -fsSL https://pkgs.tailscale.com/stable/raspbian/bullseye.noarmor.gpg | sudo tee /usr/share/keyrings/tailscale-archive-keyring.gpg > /dev/null
curl -fsSL https://pkgs.tailscale.com/stable/raspbian/bullseye.tailscale-keyring.list | sudo tee /etc/apt/sources.list.d/tailscale.list

sudo apt-get update
sudo apt-get install tailscale

    (If prompted, type Y and enter to confirm installation.)

  3. To connect your Pi to your Tailscale network, run

    sudo tailscale up

    This will result in a prompt with a URL link to click. Follow that link by command-clicking it or pasting it into your browser! It will ask you to agree to connect this new device to your Tailscale network; follow through with whatever verification steps it asks you to do.

    Note that if you want to connect the device to someone else‚Äôs Tailscale network, you‚Äôll need to send them the generated URL and have them authorize it with their own Tailscale account.

  4. Your Pi should now show up on your Tailscale console.

  5. If you won‚Äôt have easy access to the device in the future, consider setting ... > Disable key expiry on that Pi in the Tailscale web console.

  6. Optional: If this is your first time using Tailscale, you‚Äôll need to additionally install it on whatever device you plan to use to access the BirdNET-Pi‚Äîe.g., I have it installed on both my personal computer and my phone. Steps for these different devices are available in their documentation.

    ssh <pi_username>@birdnet-<pi_identifier>

    Note the lack of .local at the end of the host name! E.g. over Tailscale I now do ssh birbwatcher@birdnet-hannex instead of ssh birbwatcher@birdnet-hannex.local.

  8. Similarly, you can view the Pi‚Äôs website via your Tailscale network, so that you no longer have to be on the same wireless network as the Pi! You‚Äôll still need to do this from a machine that is part of your Tailscale network‚Äîthis url won‚Äôt work from a machine that isn‚Äôt, i.e., you can‚Äôt just give the URL to a friend and expect it to work.

    Find the Pi‚Äôs URL in your Tailscale console. E.g., for my device, the console shows its address as

    so I access mine at http://birdnet-hannex.feist-matrix.ts.net/. If you go to that address, you will not be able to see anything‚Äîbecause you are not part of my Tailscale network.

That‚Äôs it!

While SSH‚Äôd onto the Pi, you can do sudo tailscale down to disconnect it from the VPN, or sudo tailscale up to reconnect. (tailscale status will tell you whether you are currently connected or not.)

System administration details

Your BirdNET-Pi is now running! Awesome! Congrats! Uh‚Ä¶what if something goes wrong?

  • If your power goes out, or your Wi-Fi router resets, the Pi will automatically reconnect and start back up.
  • If your Pi is unable to connect to the Wi-Fi for whatever reason, BirdNET will continue to run its detection and analysis; you just won‚Äôt be able to connect to it (i.e., see the website for your installation) until the Pi has connected to the Wi-Fi again.
  • If your internet is out but your Wi-Fi router is still running, you‚Äôll still be able to see your Pi‚Äôs website
  • If you are unable to load your Pi‚Äôs website from another device, check the following (in roughly this order):
    1. Is the Pi plugged into power? Is the green light on it flashing (to indicate that it is powered)?
    2. Is the Pi able to access the wireless network it was configured to run on? i.e., is that network accessible and can you connect to it with another device (i.e., your phone, a laptop, etc)?
