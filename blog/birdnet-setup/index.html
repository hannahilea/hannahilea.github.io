Task 0: Acquire components

Here are the items you‚Äôll need, with prices at time of writing:

  • Raspberry Pi Zero 2 W [ Adafruit, $15 | MicroCenter, $18]

  • USB lavalier microphone [ Amazon, $19 plus USB 2.0-to-micro adapter, $4]

Total cost: ~$60, not including the SD card writer

Task 1: Initial Raspberry Pi Zero 2 W set up

In this step, we‚Äôll install an operating system (OS) on the Raspberry Pi Zero 2 W, by loading it onto the microSD card.

  1. Plug the SD card writer into your computer, and put the microSD card into it. (The microSD card almost certainly came with an adapter that allows it to fit into the SD card writer slot.)

  2. Install the Raspberry Pi Imager application on your computer, by selecting the correct ‚ÄúDownload for Windows/macOS‚Äù option on this page: Raspberry Pi Imager

  3. Open the Raspberry Pi Imager application, and make the following selections:

    • Raspberry Pi Device: Raspberry Pi Zero 2 W
    • Operating System: Raspberry Pi OS (Legacy, 64-bit) LITE
      • This option may be nested in the ‚ÄúRaspberry Pi OS (other)‚Äù folder and require a little bit of searching.
    • Storage: Generic - SD/MicroSD Media, which is the microSD card that‚Äôs currently in the writer

    ‚Ä¶NEXT!

  4. You‚Äôll be prompted to optionally ‚ÄúApply OS customization settings?‚Äù, for which you should select EDIT SETTINGS. On this page, set:

    • Hostname: birdnet-<pi_identifier>,
      • This will be the name of the Raspberry Pi, e.g., birdnet-foo, birdnet-backyard, birdnet-007, whatever! The identifier I chose in these screenshots is hannex, such that my full hostname is birdnet-hannex.
      • Make it something unique enough that if you set up multiple birdnets, you‚Äôll be able to keep them separate. I typically use the initials of the recipient.
      • Note that the angle brackets (‚Äú<‚Äù, ‚Äú>‚Äù) here and elsewhere in this tutorial should not be included in your chosen <pi_identifier>; they are simply convention used to indicate the user-specific parts of variables.
    • Username and password: <pi_username> / <pi_password>
      • This will be the username and password for the Raspberry Pi. I use birbwatcher as the username for all of mine, so that‚Äôs what you‚Äôll see in my screenshots. Make sure to write these values down somewhere, as you‚Äôll need them whenever you want to log into your Pi!
    • Configure wireless LAN: <your_current_wifi_network_name> / <your_current_wifi_network_password>
      • Important: this must be the Wi-Fi network name (‚ÄúSSID‚Äù) and password for the Wi-Fi network your computer is currently using, and that you‚Äôll be using for the rest of the Pi setup. If you‚Äôre setting this device up for someone else, there‚Äôll be a separate step to additionally add their Wi-Fi network info later; for now, this should be yours.
    • Make sure the time zone is correct for wherever your birdnet will be installed.
    • Under the SERVICES tab, confirm that enable SSH and use password authentication are both selected‚Äîthey likely already are, by default.

    ‚Ä¶SAVE!

  5. You‚Äôll be reprompted to ‚ÄúApply OS customization settings?‚Äù; now select YES. You might get a scary looking message asking you to confirm that any existing data on the card will be overwritten; select YES to continue. If prompted for a password, use the password for the computer you‚Äôre running the configuration from‚Äînot any of the passwords we‚Äôve set for the Pi.

    The Raspberry Pi Imager will now proceed to write the operating system and configuration settings to your microSD card! It may take a few minutes, but the progress bar will tell you when its done, and the microSD card will be automatically disconnected from your computer (‚Äúejected‚Äù) once the writing is complete. At this point it is safe to remove the card from the reader!

  6. Remove the microSD card from the writer and put it into the microSD card slot in the Raspberry Pi 2 Zero W:

  7. Connect the power cable to one of the Pi‚Äôs micro USB ports, and plug its other end into the wall. Once powered, the Pi will turn on automatically; you‚Äôll be able to tell, because a green light will start flashing on the Pi. This may take a few moments!

The Pi will use the Wi-Fi credentials you set earlier to automatically connect to your wireless network. We can (and will!) do the rest of the setup wirelessly!

Task 2: SSH time‚Äîtalk to the Pi

This step is barely worthy of its own section, but giving it a header should make it easier for you to find again when you inevitably forget how to connect. :D

By choosing the ‚ÄúLite‚Äù operating system in the previous task, we configured the Pi to run ‚Äúheadless‚Äù‚Äîwhich means without a dedicated monitor, keyboard, or mouse. To interact with it without those things, we connect to it remotely over the wireless network, via a protocol called SSH. As long as your personal computer is on the same Wi-Fi network you configured the Pi to connect to and the Pi is powered, you‚Äôll be able to SSH onto it.

  1. On your personal computer, open up your terminal application‚Äîif you‚Äôre on a Mac, this is literally called ‚ÄúTerminal‚Äù; on Windows it‚Äôs called ‚ÄúWindows Terminal‚Äù‚Äîand type

    ssh <pi_username>@birdnet-<pi_identifier>.local

    and then hit enter.

    For example, to SSH onto the device in my screenshots above, I type ssh birbwatcher@birdnet-hannex.local.

  2. If prompted to continue connecting, type yes and hit enter. (This will only happen the first time your computer connects to the new Pi.)

  3. When prompted for a password, type the <pi_password> from above (without the angle brackets!!) and hit enter.

That‚Äôs it, you‚Äôre on! Any commands you type into your terminal window will now be run on your Raspberry Pi. üòÆ

Oh, and if you want to close the SSH connection? Type exit and then hit enter, or just close the terminal window.

In your terminal window, how can you tell which machine you‚Äôre sending commands to? How can you tell if your SSH connection is running or has dropped?

Look at the start of each line: when you‚Äôre on the Pi, the ‚Äúprompt‚Äù‚Äîthe default text at the start of the line in your terminal‚Äîwill be <pi_username>@birdnet-<pi_identifier>. When you‚Äôre on your personal machine, it will be something else, probably the name of your computer unless you‚Äôve customized it to look different.

In the screenshot, my local prompt is the path to my working directory ‚Äú~‚Äù in a blue arrow, and the prompt on the Pi is the white birdnet-hannex. (The text styling may look different for yours, but the concept is the same.)

If your SSH connection drops at any time‚Äîyour internet chokes, you decide it‚Äôs bedtime and want to finish another day, you accidentally close your terminal window‚Äîsimply SSH back on again.

Task 3: BirdNET-Pi application installation

Okay. You‚Äôve set up the Pi, you‚Äôve ssh‚Äôd onto the Pi‚Äîit‚Äôs time to add the ‚ÄúBirdNET‚Äù part of BirdNET-Pi!

In this step, we‚Äôll download and run the installer for the BirdNET-Pi application, and then change some of the Pi‚Äôs configuration for storage limits, restart behavior, and logging. Some of the following is going to seem a little silly, but it works. ¬Ø\_(„ÉÑ)_/¬Ø Bear with me!

If you‚Äôre prompted for a password at any point, use the <pi_password> you set previously. When a step says ‚Äúrun‚Äù, that means to type or copy the given text into the terminal and then hit enter.

  1. In your remote terminal‚Äîthat is, the one where you‚Äôve SSH‚Äôd onto your Pi‚Äîrun sudo nano /etc/rc.local

    This will open up a new file (which will be saved to /etc/rc.local) in a program called nano, which is a text editor that runs in the terminal. Copy and paste the following into it: #!/bin/sh -e iw wlan0 set power_save off exit 0 Select ctrl and x at the same time to close the file, then Y followed by enter to save this content to the new file. After the file saves you‚Äôll be returned back to your command prompt.

    Want to confirm that you write and saved the correct content to this file? Run cat /etc/rc.local to print the contents of that /etc/rc.local file in the terminal. If the content matches the above text, you‚Äôre good to go‚Äîif it doesn‚Äôt, simply try again: do sudo nano /etc/rc.local to reopen the file in your editor, and edit it so that it matches!

  2. Run sudo chmod u+x /etc/rc.local in the terminal. This makes the new file you just created executable, i.e., allows the computer to run it.

  3. Update the swap space limits! Run sudo nano /etc/dphys-swapfile to open the file /etc/dphys-swapfile in nano. Then press the down/up arrow keys to find and update the lines that contain CONF_SWAPSIZE and CONF_MAXSWAP. For each line, edit it to:

    1. Delete the # from the start of the line, if present, and
    2. Update their values to be CONF_SWAPSIZE=2048 and CONF_MAXSWAP=4096, respectively.

    This is what part of that file should look like, after you‚Äôve edited it:

    A # at the start of the line means that the line is commented out‚Äîwhich means that the changes on that line will be ignored. We do not want them to be ignored, so we delete that #!

    Once more, hit ctrl and x together, followed by Y, followed by enter, to save these changes; once more, you‚Äôll be returned to the terminal.

  4. In the terminal, run sudo reboot. This will cause the Pi to‚Äîyou guessed it‚Äîreboot, which will automatically close your ssh connection.

  5. Wait for a moment or two for the Pi to reboot, then SSH back onto it:

    ssh <pi_username>@birdnet-<pi_identifier>.local

    (But you knew that already by now, right? :) )

  6. Okay, this is the bit that is going to feel silly. Just go with it. In your remote terminal, run

    curl -s https://raw.githubusercontent.com/Nachtzuster/BirdNET-Pi/main/newinstaller.sh | bash

    A bunch of text will print out to the screen, but you don‚Äôt have to pay attention. Eventually, it WILL eventually fail, with a ‚ÄúNo space left on device‚Äù error. That‚Äôs expected! You want it to happen!

  7. After you‚Äôve gotten the error, run the following at the prompt:

    sudo mount -o remount,size=5G /tmp/
rm -rf BirdNET-Pi/
curl -s https://raw.githubusercontent.com/Nachtzuster/BirdNET-Pi/main/newinstaller.sh | bash

    ‚Ä¶when it eventually succeeds (eventually! This step is slow, lots of things happen during installation) the Pi will reboot.

  8. ‚Ä¶SSH back onto it again! I bet you remember the command this time.

    You can also probably use the up-arrows in your terminal to page back through the previous command history in your terminal, which at this point definitely contains your SSH command! You can hit enter on any item in the command history to rerun it.

  9. Run sudo nano /etc/systemd/journald.conf which will dump you into yet another file in your text editor!

    • Replace #SystemMaxUse= with SystemMaxUse=200M and then

    • Save the file ( ctrl+x , Y, enter!).

  10. Run systemctl restart systemd-journald. This will prompt you to enter your password <pi_password> when prompted.

At this point, you‚Äôve installed the BirdNET-Pi application! Congrats!

Task 4: Configure the BirdNET-Pi application

The BirdNET-Pi application consists of two components: a classification pipeline that constantly analyzes recorded audio for bird sounds, and a small web server that hosts a website to visualize those detections. The website also lets users configure the application‚Äôs various settings. That‚Äôs what we‚Äôll be doing in this task!

For the following steps, use whatever browser on your computer you prefer: Chrome, Firefox, whatever.

  1. In your browser of choice, go to the website http://birdnet-<pi_identifier>.local/](http://birdnet-<pi_identifier>.local/). (For my installation, that's [http://birdnet-hannex.local/). Note that this site will only work if you are visiting it from a machine on the same Wi-Fi network that your Pi is currently connected to!

    You should see something like this, only with no detections yet:

  2. Select the Tools tab. When prompted for a username and password, type birdnet and leave the password empty.

    • This is the default username and password of the BirdNET-Pi application, which we have not yet set! If you want to change it, you can do that on the settings page; I haven‚Äôt bothered to change mine because I trust anyone who I‚Äôve granted access to my Wi-Fi network not to mess with my BirdNET-Pi settings.

  3. In Tools > Services:

    1. Disable the Live Audio Stream. This is optional‚Äîyou could leave it enabled. I disable mine for two reasons:
      1. I don‚Äôt want the ability to hear the recorded audio stream live, so it is a waste of compute power to leave it on, potentially slowing everything else down; and
      2. It makes it too easy to accidentally is a potential privacy violation, as the live audio hasn‚Äôt been analyzed yet so doesn‚Äôt bypass/throw out any human sounds.
    2. Disable the Streamlit Statistics. This service the Species Stats tab, which is unusably slow on the Raspberry Pi Zero. Might as well turn it off, since we can‚Äôt use it!

    For both these services, be sure to select Disable instead of Stop; the former will persist after reboot, the latter will only halt the service until reboot.

    Note that it may take a few seconds to a minute for these settings to update after being clicked; your Pi is working very hard to analyze birds and handle website requests at the same time! Have patience.

  4. In Tools > Settings, update latitude and longitude to that of your planned installation location, then select Update settings.

    How do you find that info? In Google Maps, right click on a location to see its latitude and longitude:

  5. In Settings > Advanced Settings, set:

    • Privacy threshold to 1%;
    • Recording Length to 30 then select Update settings.

Task 5 (Optional): Install Tailscale for remote monitoring

If you‚Äôre gifting this birdnet to someone else who wants you to act as tech support for their installation, or you want to be able to check on your own installation while you‚Äôre not connected to your home Wi-Fi network, you can add your Pi to a Tailscale network‚Äîbasically, it acts as a virtual private network for your various devices.

Click to see instructions

  1. Sign up for a Tailscale account, if you don‚Äôt already have one.

  2. In your terminal, while SSH‚Äôd onto the Pi, install Tailscale by typing/copying:

sudo apt-get install apt-transport-https

curl -fsSL https://pkgs.tailscale.com/stable/raspbian/bullseye.noarmor.gpg | sudo tee /usr/share/keyrings/tailscale-archive-keyring.gpg > /dev/null
curl -fsSL https://pkgs.tailscale.com/stable/raspbian/bullseye.tailscale-keyring.list | sudo tee /etc/apt/sources.list.d/tailscale.list

sudo apt-get update
sudo apt-get install tailscale
  1. To connect your Pi to your Tailscale network, type
sudo tailscale up

This will result in a prompt for what to do next, with a linked URL to click. Follow that prompt! It will basically just ask you to agree that you want to connect this new device to your Tailscale network.

- Note that if you want to connect the device to *someone else's* Tailscale network, you'll need to send *them* the generated URL and have them authorize it with their Tailscale network.

  1. Once you‚Äôve followed that prompt, your new device should now show up on your Tailscale console.

  2. If you won‚Äôt have easy access to the device in the future, consider setting ... > Disable key expiry on that Pi in the console.

Task 6 (Optional): Support additional Wi-Fi networks

There may be multiple Wi-Fi networks to which you want the Pi to auto-connect when in range. For example, when I set up a birdnet for someone else, I want it to autojoin my home network‚Äîfor quality checking before I send it off!‚Äîas well as the network of the home I‚Äôm sending it to. Perhaps you want to allow your Pi to connect to your phone‚Äôs hotspot, so that you can take it with you on various outdoor adventures?2

Click to see instructions

For each additional network you want to add, paste the following:

  1. Type the following:

    sudo nmcli connection add \
    type wifi \
    wifi.ssid "<NETWORK-NAME>" \
    wifi-sec.key-mgmt wpa-psk \
    wifi-sec.psk "<NETWORK-PASSWORD>"

    e.g., for a network named SuperBoringNetwork381 with password reallysecurepassword:

    sudo nmcli connection add \
    type wifi \
    wifi.ssid "SuperBoringNetwork381" \
    wifi-sec.key-mgmt wpa-psk \
    wifi-sec.psk "reallysecurepassword"

  2. Reload the configuration: sudo nmcli connection reload

  3. ‚Ä¶that‚Äôs it! You can check to see what you‚Äôve set up with nmcli device wifi list

For now, you can close your terminal window to close your SSH connection if you want‚Äîwe won‚Äôt need it anymore.

System administration details

Your BirdNET-Pi is now running! Awesome! Congrats! Uh‚Ä¶what if something goes wrong?

  • If your power goes out, or your Wi-Fi router resets, the Pi will automatically reconnect and start back up.
  • If your Pi is unable to connect to the Wi-Fi for whatever reason, BirdNET will continue to run its detection and analysis; you just won‚Äôt be able to connect to it (i.e., see the website for your installation) until the Pi has connected to the Wi-Fi again.
  • If your internet is out but your Wi-Fi router is still running, you‚Äôll still be able to see your Pi‚Äôs website
  • If you are unable to load your Pi‚Äôs website from another device, check the following (in roughly this order):
    1. Is the Pi plugged into power? Is the green light on it flashing (to indicate that it is powered)?
    2. Is the Pi able to access the wireless network it was configured to run on? i.e., is that network accessible and can you connect to it with another device (i.e., your phone, a laptop, etc)?
    3. (If you aren‚Äôt running tailscale) Is the device you‚Äôre trying to view the website from connected to the same network as the device is? This is the gotcha question‚Ä¶if they aren‚Äôt on the same network, the website won‚Äôt be viewable. (This means if your phone is on the 5g home network and your Pi is on the regular network, you won‚Äôt be able to view the website.)
    4. (If you are running tailscale) Does the device you‚Äôre trying to view the website from have tailscale enabled? Are you loading the site through its tailscale address (TODO) rather than FOO.local (TODO)?
    5. Can you connect to the Pi via ssh? If the Pi is powered and connected to the Wi-Fi,
  • If you set the device up for someone else and it turns out you set up their Wi-Fi password incorrectly, use your phone to create a hotspot with the same credentials of the Wi-Fi network you were on when you first set up the Pi (i.e., your home network, if you set this up for someone else) to get onto the Pi.4
  • Once the Pi begins to run out of storage space, which won‚Äôt happen for a long time,5 the oldest recordings will be deleted to make space for newer recordings.
    • You can preempt this ‚Äúloss‚Äù by periodically moving older recordings off the device to a different storage location.
